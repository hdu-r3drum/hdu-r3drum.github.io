LeetCode刷题记录 [Hard]Median of Two Sorted Arrays

Edwards Lv2

  • LeetCode 
 

Description

Given two sorted arrays nums1 and nums2 of size m and n respectively, return the median of the two sorted arrays.
The overall run time complexity should be O(log (m+n)).

分析

一种很直接的思路就是开辟一个新的长度为n+m的数组，然后取中间值（如果为奇数个元素即为size / 2，偶数计算size - 1 / 2和size / 2的平均数）

显然，这种方法时间复杂度为O(m+n)
有什么优化方法呢？题目要求O(log(m+n))的时间复杂度，这提示可能需要用二分查找的方法。因为二分查找的时间复杂度是O(log n)级别的。那如何将二分查找应用到这个问题中呢？
对于两个有序数组找中位数，可以将问题转化为寻找第k小的元素，其中k是合并后的中间位置。例如，总长度是m+n，如果总长度是奇数，k就是(m+n+1)/2；如果是偶数，则需要找到第k和k+1的两个数，取平均。
问题就转化为如何高效地找到两个有序数组中的第k小元素。这时候可以利用二分查找每次排除掉一部分不可能的元素，从而将时间复杂度降到O(log(m+n))。

Submission

核心思路解析

问题转换

将中位数问题转化为寻找第k小元素的问题。对于长度为奇数的合并数组，中位数是第k=(m+n+1)/2小的元素；对于偶数，则是第k=(m+n)/2和k+1两个元素的平均值

二分查找策略

  • 每次比较两个数组的第k/2个元素（若存在），排除较小元素所在数组的前k/2部分。
  • 若某数组剩余长度不足k/2，则取其末尾元素进行比较，并排除该数组剩余的所有元素

边界处理

  • 当某一数组被完全排除时，直接从另一数组的当前起始位置取第k小元素。
  • k=1时直接比较两个数组当前的最小元素。
