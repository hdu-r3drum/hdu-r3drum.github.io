Java 泛型和通配符

Edwards Lv2
  2025-03-13 15:34:32 2025-03-13 15:34:32 创建   2025-03-13 15:49:18 2025-03-13 15:49:18 更新  
  • Java学习 
 

泛型中的<T>是类型参数，通常用于类、接口或方法的声明中，表示一个具体的类型，可以在代码中实际使用。而<?>则是通配符，用于更灵活的类型匹配，通常在方法参数或变量声明中使用，表示未知类型。

前言

  • List<T> 是泛型类的具体实例化（如 List<String>）
  • List<?> 表示一个未知类型的 List，无法直接写入（list.add(...) 会编译报错）
  • 不能相互替代，<T> 用于定义具体类型，而 <?> 用于接受未知类型，二者用途不同。

类型参数（Type Parameter）

定义

  • <T> 是一个具体的类型变量，用于声明泛型类、接口或方法中的类型参数。

特点

  • 明确指定类型：T 在代码中可以作为一个具体的类型被引用（如方法的参数、返回值、变量类型等）。

  • 需要显式声明：必须在类、方法或接口的泛型声明中定义（如 class Box<T> { ... }）。

  • 允许类型操作：可以对T进行类型约束（如 <T extends Number>）或直接使用T 类型的方法。

通配符（Wildcard）

定义

  • <?> 表示未知类型，用于泛型代码中接受不特定类型的参数，提供更灵活的类型匹配。

特点

  • 类型未知：无法直接引用 ? 的具体类型（如不能声明 List<?> list = new ArrayList(); 后直接操作 list.add(“abc”)）。

  • 无需声明：直接在代码中使用（如方法参数 void printList(List<?> list)）。

  • 支持上下界：<? extends T>：接受 T 或其子类（生产者场景，只能读取）。<? super T>：接受 T 或其父类（消费者场景，可以写入）。

对比

特性 <T> <?>
类型明确性 具体类型（如 String、Integer） 未知类型（只能通过上下界约束）
使用场景 泛型类、方法、接口的定义 方法参数、变量声明（接受未知类型）
能否直接操作类型 可以（如 T value = getValue()） 不能（需通过 Object 或上下界）
类型约束 支持 支持 <? extends SomeClass>
是否允许写入数据 允许（类型已知） 通配符无界时不允许写入（类型未知）

使用场景

<T> 适用场景：

  • 需要定义泛型类、方法或接口时。

  • 需要在代码中直接操作具体类型（如返回值、参数类型）。

  • 需要对类型进行明确的约束（如 <T extends Comparable<T>>）

<?> 适用场景：

  • 需要接受不特定类型的泛型参数（如工具类方法）。

  • 需要灵活的类型匹配（如 List<? extends Number> 接受 List<Integer> 或 List<Double>）。

  • 需要限制对泛型容器的写入操作（如使用 <?> 的集合只能读取）。
