快速失败机制（Fail-Fast）

迭代器内部维护 modCount （修改计数器），当检测到集合在迭代过程中被意外修改（如直接调用集合的add()或 remove()），立即抛出ConcurrentModificationException。

关键逻辑

  • 快速失败：通过比较 modCount（集合的修改次数）与 expectedModCount（迭代器记录的初始修改次数），若不一致则说明集合被意外修改。

迭代器性能对比

集合类型 迭代器实现方式 时间复杂度 适用场景
ArrayList 数组索引游标 O(1) 随机访问、高频遍历
LinkedList 节点指针移动 O(1) 频繁插入/删除、顺序访问
HashSet 哈希桶遍历 O(n) 无序遍历
TreeSet 红黑树中序遍历 O(log n) 有序遍历
